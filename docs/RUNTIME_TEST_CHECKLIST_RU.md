# ZoneWars — чеклист runtime-проверки

Этот чеклист нужен перед каждым merge/release, потому что часть поведения зависит от Minecraft, Forge, TaCZ и внешних модов и не покрывается обычной Java-сборкой.

## 1. Базовая загрузка

- [ ] Клиент запускается с тем же JAR ZoneWars, что и сервер.
- [ ] Сервер стартует без crash report.
- [ ] В логах есть строка регистрации мода ZoneWars.
- [ ] `/zw state` отвечает без ошибки.

## 2. TaCZ

- [ ] В `logs/latest.log` есть `Registered TaCZ Forge event`.
- [ ] Нет `Could not register TaCZ event`.
- [ ] `/zw validatekits` показывает, что оружие и патроны собираются.
- [ ] После `/zw kit ak101` игрок получает оружие и патроны.
- [ ] Выстрелы до старта матча блокируются.
- [ ] Урон по союзнику блокируется.
- [ ] Урон по противнику засчитывается в hit marker и статистику.
- [ ] Убийство попадает в kill feed.

## 3. Команды и матч

- [ ] `/zw join red` работает.
- [ ] `/zw join blue` работает.
- [ ] `/zw join auto` распределяет команду.
- [ ] `/zw join garbage` не крашит команду и показывает ошибку.
- [ ] `/zw start` переводит матч в PREPARING.
- [ ] После подготовки матч переходит в ACTIVE.
- [ ] `/zw stop` сбрасывает арену.

## 4. Точки захвата

- [ ] Встать на точку после старта матча.
- [ ] Выполнить `/zwa capturedebug`.
- [ ] Проверить `world=true`.
- [ ] Проверить `inside=true`.
- [ ] Статус становится `CAPTURING`.
- [ ] Прогресс доходит до `100`.
- [ ] Статус становится `OWNED`.
- [ ] Счёт команды начинает расти.
- [ ] Если игрок вышел из зоны, прогресс не растёт.
- [ ] Если две команды стоят на точке поровну, статус `CONTESTED`.

## 5. Экономика и магазин

- [ ] `/zw balance` показывает стартовый баланс.
- [ ] `/zw shop` открывается только около сохранённой точки магазина, если такие точки есть.
- [ ] Покупка оружия списывает деньги.
- [ ] Покупка при недостаточном балансе не выдаёт предмет.
- [ ] `medkit` при полном здоровье не списывает деньги.
- [ ] Захват точки даёт награду.
- [ ] Убийство даёт награду.

## 6. Респаун

- [ ] `/zw respawn confirm` не телепортирует живого игрока.
- [ ] После смерти появляется prompt респауна.
- [ ] Респаун на базу работает.
- [ ] Палатка ставится только не рядом с базой, точкой и врагом.
- [ ] Аванпост ставит только лидер squad.
- [ ] Вражеский игрок может ломать палатки/аванпосты.
- [ ] Союзник не может ломать свой объект.

## 7. Squads и clans

- [ ] `/squad create` создаёт отряд.
- [ ] Лидер может пригласить игрока своей команды.
- [ ] Игрок другой команды не приглашается через команду.
- [ ] `/sc message` видят только участники отряда.
- [ ] `/clan create TAG COLOR` создаёт клан.
- [ ] `/cc message` видят только участники клана.

## 8. Сохранения

- [ ] После изменения арены создаётся `config/zonewars/arena.json`.
- [ ] После последующей записи появляется `arena.json.bak`.
- [ ] При повреждённом основном JSON сервер пробует восстановиться из `.bak`.
- [ ] `player_stats.json` обновляется после окончания матча.
- [ ] `matches.json` хранит историю матчей.

## 9. Клиентский UI

- [ ] HUD не вызывает сильного FPS drop после join.
- [ ] Карта открывается на `M`.
- [ ] Инвентарь открывается на `I`.
- [ ] Squad UI открывается на `O`.
- [ ] Danger ping работает и не спамится бесконечно.
- [ ] На карте не видны точные координаты противника.

## 10. Перед релизом

- [ ] `./gradlew clean test` успешен.
- [ ] `./gradlew clean build` успешен или успешен GitHub Actions Build.
- [ ] Проверен JAR из `build/libs` или artifact GitHub Actions.
- [ ] Версия в `gradle.properties` обновлена при необходимости.
